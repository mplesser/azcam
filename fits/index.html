    Fits Commands

    This class provides FITS image support functions.

    Usage Example:

    rois = azcam.fits.col_bias("test.fits")

    azcam.fits

    Contains FITS image manipulation commands.

    add(filename1, filename2, filename3, datatype='uint16')

    Add two images.

    Parameters:

    Name Type Description Default
    filename1 str

    image filename

    		 required
    filename2 str

    may be an image filename or a constant.

    		 required
    filename3 str

    optional, must be an image filename. If not specified, result goes into filename1.

    		 required
    datatype str

    valid datatype string for resultant data type

    		 'uint16'

    Returns:

    Type Description
    None

    None

    add_history(filename, history_string, extension=0)

    Adds a HISTORY line containing 'history_string' to the image. Used to record actions performed on an image. A 20 character time stamp is added as a prefix, and the result is split across up to three cards if it is too long to fit in one. Any extra text is truncated.

    filename: image filename history_string: string to add as a HISTORY keyword extension: image extension number

    arith(filename1, operator, filename2, filename3='', datatype='uint16')

    Simple image manipulation of FITS files. Converts back to unsigned 16 bit.

    filename1: image filename operator: '+','-','/','*' filename2: may be an image filename or a constant. filename3: optional, must be an image filename. If not specified, result goes into filename1. datatype: valid datatype string for resultant data type

    colbias(filename='test', fit_order=3, margin_cols=0)

    Remove column bias from a FITS file.

    filename: image filename fit_order: polynomial fit order, use 0 to remove median not fitted value margin_cols: number of overscan columns to skip before correction

    combine(file_list=[], out_filename='combined.fits', combination_type='median', overscan_correct=1, fit_order=3)

    Make a combination of a list of FITS filenames.

    file_list: list of filenames to combine out_filename: output filename combination_type: combination type, "median", "sum", or "mean" overscan_correct: > 0 to correct overscan with specified fir order before combination

    div(filename1, filename2, filename3, datatype='uint16')

    Divide Image1 by Image2.

    filename1: image filename operator: '+','-','/','*' filename2: may be an image filename or a constant. filename3: optional, must be an image filename. If not specified, result goes into filename1. datatype: valid datatype string for resultant data type

    edit_keyword(filename, keyword, value, extension=0)

    Edits a header keyword value.

    filename: image filename keyword: keyword name value: new value of keyword extension: image extension number

    file_exists(filename)

    Parameters:

    Name Type Description Default
    filename str

    filename to check if exists

    		 required

    Returns:

    Type Description
    bool

    True if the file exists

    get_all_data(filename='test')

    Return all pixel data from image.

    filename: image filename

    get_data(filename='test', roi=[])

    Return data (pixel values) from an ROI in an image for every extension. NOT FINISHED!

    filename: image filename roi: ROI

    get_extensions(filename)

    Returns [NumExt,first_ext,last_ext] which are the number of IMAGE extensions and extension number limits FOR THE RANGE command [range(first_ext,last_ext)]. NumExt is 0 for a standard FITS file and >0 for MEF. The first data extension for an MEF file is 1.

    filename: image filename

    get_header(filename, extension=0)

    Return the image header as a pyfits header object.

    filename: image filename extension: image extension number

    get_history(filename, extension=0)

    Returns the HISTOR header lines.

    filename: image filename extension: image extension number

    get_keyword(filename, keyword, extension=0)

    Return a header keyword value.

    filename: image filename keyword: keyword name extension: image extension number

    get_section(filename, section, extension=0)

    Extract image section pixel values from a FITS keyword to a list. Format is always zero-based: [first_col,last_col,first_row,last_row].

    filename: image filename section: section name (like "CCDSEC") extension: image extension number

    maximum(filename='test', roi=[])

    Compute maximum of an image ROI for every extension.

    filename: image filename roi: ROI

    mean(filename='test', roi=[])

    Compute mean of an image ROI in every extension.

    filename: image filename roi: ROI

    minimum(filename='test', roi=[])

    Compute minimum of an image ROI for every extension.

    filename: image filename roi: ROI

    mult(filename1, filename2, filename3, datatype='uint16')

    Multiple two images.

    filename1: image filename operator: '+','-','/','*' filename2: may be an image filename or a constant. filename3: optional, must be an image filename. If not specified, result goes into filename1. datatype: valid datatype string for resultant data type

    resample(filename='test', resample=2)

    Resample an image by combining adjacent pixels.

    filename: image filename resample: number of pixels to combine in each dimension

    sdev(filename='test', roi=[])

    Compute standard deviation of an image ROI in every extension.

    filename: image filename roi: ROI

    stat(filename='test', roi=[])

    Compute mean and sdev image statistics of ROI in every extension. Returns [[means], [sdevs], ROI] filename: image filename roi: ROI

    sub(filename1, filename2, filename3, datatype='uint16')

    Subtract filename2 from filename1.

    filename1: image filename operator: '+','-','/','*' filename2: may be an image filename or a constant. filename3: optional, must be an image filename. If not specified, result goes into filename1. datatype: valid datatype string for resultant data type
